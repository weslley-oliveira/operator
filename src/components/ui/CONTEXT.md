# UI COMPONENTS — CONTEXT & CONVENTIONS

GENERIC AND REUSABLE COMPONENTS SHARED ACROSS PAGES. THE REFERENCE DESIGN COMES FROM PENCIL (`.PEN` FILES).

## STACK

| LIB | USAGE |
|---|---|
| **TAILWIND CSS V4** | STYLING VIA UTILITY CLASSES |
| **TAILWIND-VARIANTS** (`TV`) | COMPONENT VARIANT DEFINITION |
| **TAILWIND-MERGE** (`TWMERGE`) | SAFE CLASS MERGING ON THE FINAL `CLASSNAME` |

## COMPONENT STRUCTURE

```tsx
import type { ComponentProps } from 'react';
import { twMerge } from 'tailwind-merge';
import { tv, type VariantProps } from 'tailwind-variants';

// 1. DEFINE VARIANTS WITH tv()
const component = tv({
  base: ['...BASE CLASSES...'],
  variants: {
    variant: { primary: '...', secondary: '...' },
    size:    { sm: '...', md: '...', lg: '...' },
  },
  defaultVariants: {
    variant: 'primary',
    size: 'md',
  },
});

// 2. TYPE BY EXTENDING THE NATIVE HTML ELEMENT
type ComponentVariants = VariantProps<typeof component>;
type ComponentProps = ComponentProps<'button'> & ComponentVariants;

// 3. NAMED EXPORT — NEVER DEFAULT EXPORT
export function Component({ variant, size, className, ...props }: ComponentProps) {
  return (
    <element
      {...props}
      className={twMerge(component({ variant, size }), className)}
    />
  );
}
```

## RULES

- **NAMED EXPORTS** ALWAYS — `EXPORT FUNCTION FOO`, NEVER `EXPORT DEFAULT`.
- **EXTEND NATIVE PROPS** VIA `COMPONENTPROPS<'ELEMENT'>` TO NOT BLOCK HTML/ARIA ATTRIBUTES.
- **`CLASSNAME` ALWAYS MERGEABLE** — THE CONSUMER CAN OVERRIDE CLASSES VIA `TWMERGE`.
- **`TV()` OUTSIDE THE COMPONENT** — INSTANTIATE AT MODULE SCOPE, NOT INSIDE RENDER.
- **`SIZE`** VARIANTS USE THE `SM / MD / LG` SCALE; **`VARIANT`** DESCRIBES THE VISUAL INTENT.

## TYPOGRAPHY

- MONO FONT: **JETBRAINS MONO** (CONFIGURED VIA `NEXT/FONT` IN `LAYOUT.TSX` AND EXPOSED AS `--FONT-MONO` IN `GLOBALS.CSS`).
- USE `FONT-MONO` IN COMPONENTS THAT FOLLOW THE DESIGN SYSTEM.
